## 先搞懂几个概念

听不懂这些名词没关系，先有个印象就行，后面照着做就好：

| 名词 | 大白话解释 |
|------|-----------|
| **Git** | 一个"版本管理工具"，能帮你记住文件每次改了什么。我们要用它来上传和下载笔记。 |
| **GitHub** | 一个"存代码的网站"，你可以把笔记放在上面，像网盘一样，但它和 Git 配合得更好。 |
| **仓库（Repository）** | 就是 GitHub 上存放你笔记的那个项目文件夹。比如你的笔记仓库地址是 `https://github.com/CCCpc0/notes`。 |
| **Git Bash** | 一个黑色的命令行窗口，我们在这个窗口里敲命令来操作 Git。后面所有命令都在这里面输入。 |
| **克隆（clone）** | 把 GitHub 上的笔记仓库"复制一份"到你的新电脑上。 |
| **拉取（pull）** | 从 GitHub 上把别人（或另一台电脑）更新的内容下载到当前电脑。 |
| **提交（commit）** | 告诉 Git："我改完了，帮我保存一下这次的修改记录。" |
| **推送（push）** | 把本地保存好的修改，上传到 GitHub 网站上。这样其他电脑才能拿到最新内容。 |

## 场景一：新电脑第一次设置（只做一次）

当你换了新电脑，或者重装了系统，按以下步骤操作。

### 第 1 步：下载安装 Git

1. 打开浏览器，输入这个网址：https://git-scm.com/download/win
2. 网页会自动弹出下载，下载完成后双击打开安装包
3. 安装过程中 **一路点"下一步（Next）"** 即可，所有选项保持默认，不用改任何东西
4. 安装完成后，在键盘上按 `Win 键`（键盘左下角 Windows 图标），输入 `Git Bash`，看到图标后点回车打开

> ✅ 出现一个黑色窗口，里面有光标闪烁，说明安装成功。

### 第 2 步：告诉 Git 你是谁

在 Git Bash 黑色窗口里，依次输入以下两行命令，每输完一行按一次回车：

```bash
git config --global user.name "CCCpc0"
```

```bash
git config --global user.email "455339599@qq.com"
```

> 💡 这两行是告诉 Git "这台电脑是 CCCpc0 在操作"。输入正确的话不会有任何提示，这是正常的。
>
> 📝 输入方法提示：可以复制上面的命令，在 Git Bash 窗口中按 `Shift + Insert` 粘贴（Ctrl+V 在这里不管用）。

### 第 3 步：把笔记下载到电脑

先想好你要把笔记放在哪个文件夹，比如放在 `D盘` 的 `学习` 文件夹里。

在 Git Bash 中输入：

```bash
cd /d/学习
```

> 💡 `cd` 就是"进入某个文件夹"的意思。`/d/学习` 就是 `D:\学习` 文件夹。

然后输入下面这行命令，把 GitHub 上的笔记下载下来：

```bash
git clone https://github.com/CCCpc0/notes.git
```

> ✅ 看到 `done` 字样说明下载完成。去 D 盘的学习文件夹里，就能看到一个叫 `notes` 的文件夹，里面就是你的笔记了。

## 场景二：每次写笔记前后的操作（日常使用）

### 📥 开始写笔记之前——先拉取最新内容

打开 Git Bash，进入笔记文件夹：

```bash
cd /d/学习/notes
```

然后拉取 GitHub 上最新的内容：

```bash
git pull
```

> 💡 这一步的作用是：如果你在另一台电脑上更新了笔记，这会把最新的内容同步到当前电脑。如果别人没改过，它会显示 `Already up to date.`（已经是最新的了）。

### 📤 写完笔记之后——上传到 GitHub

在 Git Bash 中依次输入以下三行命令（每行输完按回车）：

**第一步：把所有改动加入待上传列表**

```bash
git add -A
```

> 这一步是告诉 Git："我已经改好了，把这些改动打包准备上传"。输入后没有提示是正常的。

**第二步：保存改动记录**

```bash
git commit -m "记录：今天学了xxx"
```

> 这一步是给这次改动写个备注，方便以后回看。把 `xxx` 换成你今天学了什么，比如 `记录：添加了项目规划提示词`。

**第三步：推送到 GitHub**

```bash
git push
```

> 这一步才真正把笔记上传到 GitHub。看到 `100%` 就说明上传成功了。
>
> 第一次 push 时可能会弹出一个浏览器窗口让你登录 GitHub，点"授权"即可。

## 简单记忆口诀

```
写前  git pull       ← 拉下来
写后  git add -A      ← 打包
      git commit -m "xxx"  ← 备注
      git push        ← 传上去
```

## 常见问题

**Q：输入命令后提示 `fatal: not a git repository`？**

说明你当前不在笔记文件夹里。先用 `cd /d/学习/notes` 进入笔记文件夹（路径改成你自己的），再输命令。

**Q：输入 `git push` 后提示输入用户名密码？**

这是 GitHub 在验证身份。如果弹出浏览器窗口，点授权就行了。如果没弹窗，可以在 Git Bash 里输入：

```bash
git config --global credential.helper manager
```

然后重新 `git push`，会弹出 GitHub 登录窗口。

**Q：`git pull` 的时候提示有冲突怎么办？**

这说明同一份文件在两台电脑上都改过了，Git 不知道以哪个为准。这种情况比较少见，如果遇到了截图发出来，让会的人帮你解决。

**Q：我只在一台电脑上用，还需要 pull 和 push 吗？**

依然建议写完后 push。万一哪天电脑坏了或者要换电脑，GitHub 上还有一份备份，不会丢。
